## 框架概述

### 核心理念

**一次编写，多处运行** - 同一份策略代码可以在回测、模拟、实盘三种环境下运行。

### 三种运行模式

| 模式 | 枚举值 | 说明 | 适用场景 |
|------|--------|------|---------|
| **回测** | `RunMode.BACKTEST` | 历史数据回测 | 策略开发和验证 |
| **模拟** | `RunMode.SIMNOW` | CTP模拟盘 | 实盘前测试 |
| **实盘** | `RunMode.REAL_TRADING` | 真实CTP交易 | 正式交易 |

### 数据流

```
回测模式：历史数据 → 回测引擎（资金约束） → 策略 → 模拟成交
SIMNOW：  CTP行情 → 实时K线合成/data_server推送 → 策略 → 模拟下单
实盘模式：CTP行情 → 实时K线合成/data_server推送 → 策略 → 真实下单
```

### 合约代码规则

```
回测模式：
  品种+888 = 主力连续合约，用于拉取连续K线（如 au888、rb888）

SIMNOW / 实盘（自动主力映射）：
  au888  → 自动映射为当前主力月份（如 au888→au2508），用于CTP订阅和下单
  au777  → 自动映射为次主力月份
  au2508 → 指定月份，直接使用，不做映射
```

## 快速开始

### 第一个策略（3分钟）

**步骤1：创建策略文件 `my_first_strategy.py`**

```python
from ssquant.api.strategy_api import StrategyAPI
from ssquant.backtest.unified_runner import UnifiedStrategyRunner, RunMode
from ssquant.config.trading_config import get_config

def my_strategy(api: StrategyAPI):
    """简单的MA策略"""
    close = api.get_close()
    
    if len(close) < 20:
        return
    
    ma20 = close.rolling(20).mean()
    pos = api.get_pos()
    
    if close.iloc[-1] > ma20.iloc[-1] and pos <= 0:
        if pos < 0:
            api.buycover(order_type='next_bar_open')
        api.buy(volume=1, order_type='next_bar_open')
    
    elif close.iloc[-1] < ma20.iloc[-1] and pos >= 0:
        if pos > 0:
            api.sell(order_type='next_bar_open')
        api.sellshort(volume=1, order_type='next_bar_open')

if __name__ == "__main__":
    config = get_config(
        mode=RunMode.BACKTEST,
        symbol='rb888',
        start_date='2025-01-01',
        end_date='2025-11-30',
        kline_period='1h',
    )
    
    runner = UnifiedStrategyRunner(mode=RunMode.BACKTEST)
    runner.set_config(config)
    results = runner.run(strategy=my_strategy)
```

**步骤2：运行**

```bash
python my_first_strategy.py
```

> 合约参数（乘数、最小变动价、手续费等）会自动从远程服务器获取，无需手动填写。

## 策略编写

### 策略函数结构

```python
def my_strategy(api: StrategyAPI):
    """策略函数 - 每根K线执行一次"""
    # 1. 获取数据
    close = api.get_close()
    
    # 2. 数据验证
    if len(close) < 20:
        return
    
    # 3. 计算指标
    ma20 = close.rolling(20).mean()
    
    # 4. 获取持仓
    pos = api.get_pos()
    
    # 5. 交易信号
    if close.iloc[-1] > ma20.iloc[-1] and pos <= 0:
        api.buy(volume=1, order_type='next_bar_open')
```

### 初始化函数（可选）

```python
def initialize(api: StrategyAPI):
    """策略初始化 - 运行前执行一次"""
    api.log("策略初始化")
    fast = api.get_param('fast_ma', 5)
    slow = api.get_param('slow_ma', 20)
    api.log(f"参数: 快线={fast}, 慢线={slow}")

runner.run(
    strategy=my_strategy,
    initialize=initialize,
    strategy_params={'fast_ma': 5, 'slow_ma': 20}
)
```

### 资金约束（v0.4.3）

回测引擎现在执行严格的资金校验：

- **开仓校验** - 所有开仓按账户可用资金校验，资金不足自动裁剪手数
- **资金预占** - 待执行开仓单入队时预估资金占用，避免超额开仓
- **智能停止** - 资金耗尽且无持仓时提前终止，避免空跑
- **拒单可见** - 资金不足等关键信息即使 `debug=False` 也会在控制台显示

## 自动移仓

> v0.4.3 新增。仅 SIMNOW / 实盘生效，回测不支持。

主力合约换月时，框架自动完成移仓（平旧合约 → 开新主力），无需在策略里手写。

### 两种模式

| 模式 | 说明 |
|------|------|
| `simultaneous`（默认） | 同一次回调内平旧 + 开新，速度更快 |
| `sequential` | 先平旧，旧腿确认后再开新，更稳妥 |

### 配置

```python
config = get_config(
    mode=RunMode.SIMNOW,
    account='simnow_default',
    symbol='rb888',
    kline_period='15m',
    auto_roll_enabled=True,            # 开启自动移仓
    auto_roll_reopen=True,             # 平旧后自动开新（False=只平不开）
    auto_roll_mode='simultaneous',     # 移仓模式
    auto_roll_order_type='next_bar_open',
    auto_roll_log_enabled=True,        # 复盘日志
)
```

### 策略内查询

```python
def strategy(api):
    if api.is_rollover_busy():
        return  # 移仓进行中，暂停交易信号
    
    status = api.get_rollover_status()
    # ... 正常策略逻辑
```

### 完整配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `auto_roll_enabled` | `False` | 是否启用 |
| `auto_roll_mode` | `'simultaneous'` | 移仓模式 |
| `auto_roll_reopen` | `True` | 是否补开新仓 |
| `auto_roll_order_type` | `'next_bar_open'` | 下单方式 |
| `auto_roll_close_offset_ticks` | `None` | 平旧跳数 |
| `auto_roll_open_offset_ticks` | `None` | 开新跳数 |
| `auto_roll_verify_timeout_bars` | `500` | 闭环超时 |
| `auto_roll_log_enabled` | `True` | 写复盘日志 |
| `auto_roll_log_dir` | `None` | 日志目录 |
| `auto_roll_log_jsonl` | `False` | JSON Lines 格式 |

## 本地复权

> v0.4.3 新增。data_server 只存储不复权数据，复权计算在框架本地完成。

### 使用方式

```python
from ssquant.data import get_futures_data

df = get_futures_data('au888', '1D', adjust_type='1', limit=300)  # 后复权
df = get_futures_data('au888', '1D', adjust_type='2', limit=300)  # 前复权
```

### 复权类型

| 参数 | 说明 |
|------|------|
| `adjust_type='0'` | 不复权（默认） |
| `adjust_type='1'` | 后复权：最早数据保持原价，后续累积因子调整 |
| `adjust_type='2'` | 前复权：最新数据保持原价，历史数据反向调整 |

### 算法

通过 `real_symbol` 列检测合约切换点，在切换点计算比例因子 `factor = 前一根close / 切换后第一根open`，累积因子应用于 OHLC 价格列。

### 配置

`trading_config.py` 中 `ENABLE_REMOTE_ADJUST = True`（默认开启）。

## data_server模式

### 概述

除本地 CTP Tick 聚合外，框架支持 data_server 远程推送模式：

- WebSocket 实时推送任意周期K线
- 连接时自动预加载历史K线
- 断线自动重连并重新鉴权
- 订单流数据（12个字段）

### 配置

```python
config = get_config(
    mode=RunMode.SIMNOW,
    account='simnow_default',
    symbol='rb888',
    kline_period='5m',
    kline_source='data_server',       # 远程推送
    enable_tick_callback=True,        # 同时接收Tick
    tick_callback_interval=0.5,       # 节流间隔(秒)
)
```

### 节流机制（v0.4.3）

data_server + Tick回调模式下，开盘时大量Tick可能导致积压。框架内置智能节流：

- 新K线到达时立即触发策略
- 无新K线时按 `tick_callback_interval` 间隔触发
- 队列积压时自动提升间隔

| tick_callback_interval | 行为 |
|------------------------|------|
| `0.5`（默认） | 无新K线时每500ms触发一次 |
| `0` | 关闭节流，每个Tick都触发 |
| `1.0` / `2.0` | 更宽松，适合低频策略 |

## 实盘抗压机制

> v0.4.3 新增。解决行情活跃时Tick队列积压问题。

### Tick 有界队列

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `tick_queue_maxsize` | `20000` | 队列容量上限 |

| 场景 | 建议值 |
|------|--------|
| 低频/少品种 | `5000 ~ 10000` |
| 中频/10~20品种 | `15000 ~ 30000` |
| 高频/30+品种 | `30000 ~ 50000` |

### 压力等级

策略可感知底层是否进入积压状态，主动降级：

```python
def strategy(api):
    pressure = api.get_runtime_pressure()  # normal / busy / critical
    
    if api.is_runtime_under_pressure():
        return  # 高压时暂停交易
    
    stats = api.get_runtime_stats()
    # 队列长度、高水位、overflow次数、处理耗时等
```

## 常见问题 (FAQ)

### Q1: 找不到数据？

检查：
- symbol 代码是否正确（如 `rb888`）
- 日期范围是否有效
- API 账号密码是否在 `trading_config.py` 中配置

### Q2: 策略不执行？

检查：
- 数据长度是否足够（MA20 需要至少 20 根 K 线）
- 是否有早期返回（return）
- 添加 `api.log()` 调试

### Q3: 持仓不对？

检查：
- `order_type` 是否正确
- 回测用 `'next_bar_open'`，实盘用 `'market'`
- 实盘检查 `order_offset_ticks` 设置

### Q4: 回测资金不足导致开仓被拒？

v0.4.3 引入了资金约束。如果控制台显示 "资金不足" 拒单：
- 增大 `initial_capital`
- 减少开仓手数
- 检查 `margin_rate` 和 `contract_multiplier` 是否合理

### Q5: SIMNOW连接失败？

尝试：
- 更换服务器（电信1 → 电信2 → 移动）
- 检查账号密码是否正确
- 工作日 9:00-15:30 测试

### Q6: 移仓没有触发？

检查：
- `auto_roll_enabled` 是否设为 `True`
- 是否为 SIMNOW/实盘模式（回测不支持）
- 合约代码是否使用 888 格式

### Q7: data_server 开盘假死？

v0.4.3 已修复。确保 `tick_callback_interval` 不为 0（默认 0.5 秒）。多品种高频场景可适当增大到 1.0 或 2.0 秒。
