## 🔍 问题详情

### 问题描述

在平仓上海期货交易所品种时，明明有昨仓却无法平掉，CTP返回错误：

```
[ERROR] [报单错误] 错误码:50 - 平今仓位不足，请改用平昨仓
```

### 问题根因

CTP返回的`YdPosition`字段对于上期所和能源中心**不可靠**，通常返回0或错误值，导致：
1. 系统错误地认为没有昨仓（`long_yd=0` 或 `short_yd=0`）
2. 平仓时总是使用"平今"指令（CTP代码`'3'`）
3. 当实际是昨仓时，CTP返回错误50

### 修复方案

```python
# 修复前（错误）
yd_pos = data.get('YdPosition', 0)  # 上期所返回0

# 修复后（正确）
yd_pos = position - today_pos  # 数学计算，所有交易所通用
```

## 📝 详细修改

### 1. ssquant/pyctp/simnow_client.py

**位置**: `OnRspQryInvestorPosition` 方法（约第422-431行）

```python
# 【关键修复】上海期货交易所(SHFE)和能源交易中心(INE)的YdPosition字段不可靠
# 正确的昨仓计算方式：昨仓 = 总持仓 - 今仓
calculated_yd_pos = position - today_pos

if data['PosiDirection'] == '2':  # 多头
    self.client._position_cache[instrument_id]['long_today'] = today_pos
    self.client._position_cache[instrument_id]['long_yd'] = calculated_yd_pos
elif data['PosiDirection'] == '3':  # 空头
    self.client._position_cache[instrument_id]['short_today'] = today_pos
    self.client._position_cache[instrument_id]['short_yd'] = calculated_yd_pos
```

### 2. ssquant/pyctp/real_trading_client.py

**位置**: `OnRspQryInvestorPosition` 方法（约第397-406行）

修改内容与 `simnow_client.py` 相同。

### 3. ssquant/backtest/live_trading_adapter.py

1. **查看持仓日志**：策略启动时会打印持仓信息，确认昨仓数量正确

   ```
   [平多判断] au2602 多头今仓=2, 多头昨仓=3 → 需分单: 平今2手 + 平昨3手
   ```

3. **确认无错误50**：不再出现"平今仓位不足，请改用平昨仓"错误
