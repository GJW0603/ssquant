# SSQuant v0.4.2 更新日志

发布日期：2026-02-22

---

## 📡 数据服务器重大升级（重点）

本版本对远程数据服务进行了**架构级升级**，推出全新的 `data_server` 数据服务：

### 核心升级

| 功能 | 说明 |
|------|------|
| **全新架构** | 独立 data_server 服务器，更稳定、更快速 |
| **实时K线推送** | WebSocket 毫秒级推送1分钟K线，实盘无延迟 |
| **历史K线补全** | 连接时自动预加载历史K线，策略启动即有完整数据 |
| **订单流数据** | 新增12个订单流字段，支持量价分析策略 |
| **多周期支持** | 服务端直接推送任意周期K线（1M/5M/15M/1H/1D等） |

### 订单流数据字段

K线数据新增完整的订单流字段，可用于：
- 主力资金追踪
- 多空力量对比
- 量价分析策略
- 高频交易信号

```
字段        说明
────────────────────
开仓        总开仓量
平仓        总平仓量
多开        多头开仓量
空开        空头开仓量
多平        多头平仓量
空平        空头平仓量
双开        双开量（多空同时开仓）
双平        双平量（多空同时平仓）
双换        双换量（换手）
B           主买量（Buy，主动买入）
S           主卖量（Sell，主动卖出）
未知        未知方向成交量
```

### 使用示例

```python
# 获取订单流数据
klines = api.get_klines()

# 主买主卖比
buy_sell_ratio = klines['B'].iloc[-1] / (klines['S'].iloc[-1] + 1)

# 多空开仓比
long_short_ratio = klines['多开'].iloc[-1] / (klines['空开'].iloc[-1] + 1)

# 净开仓量
net_open = klines['开仓'].iloc[-1] - klines['平仓'].iloc[-1]
```

## 🚀 重大新增

### 1. Linux 平台支持

**新增 Linux CTP 二进制文件**，支持在 Linux 服务器上运行实盘交易：

每个 Python 版本目录新增 4 个 Linux 文件：
- `thostmduserapi_se.so` - CTP 行情动态库
- `thosttraderapi_se.so` - CTP 交易动态库
- `_thostmduserapi.cpython-3xx-x86_64-linux-gnu.so` - Python 绑定
- `_thosttraderapi.cpython-3xx-x86_64-linux-gnu.so` - Python 绑定

**支持的 Python 版本**: 3.9, 3.10, 3.11, 3.12, 3.13, 3.14

`ctp/loader.py` 改进：
- 自动检测操作系统类型（Windows/Linux）
- Linux 平台使用 ctypes 预加载 .so 文件
- 自动设置 `LD_LIBRARY_PATH` 环境变量

### 2. data_server K线数据源模式

新增远程数据服务器模式，支持从 data_server 获取实时 K 线推送。

**配置方式：**
```python
config = {
    'symbol': 'rb888',
    'kline_source': 'data_server',  # 新增：使用远程K线数据源
    # ...
}
```

**新增模块：**
- `ssquant/data/ws_kline_client.py` - WebSocket K线客户端
  - 支持连接 data_server WebSocket 服务器
  - 自动断线重连
  - 历史K线预加载 (preload)
  - 实时K线推送回调

- `ssquant/config/_server_config.py` - data_server 连接配置
  - ws_url: WebSocket 地址
  - api_url: REST API 地址
  - preload_count: 预加载K线数量

**新增方法 (LiveDataSource)：**
- `on_ws_kline(kline_data)` - 接收实时K线推送
- `on_ws_history(klines_list)` - 接收历史K线预加载数据

### 3. 多周期K线本地派生

新增 `ssquant/data/multi_period.py` 模块：

**实时派生器 `MultiPeriodAggregator`：**
- 从 1M K线实时派生任意周期（5M, 15M, 30M, 1H, 1D 等）
- 支持任意分钟周期（2M, 3M, 7M, 65M, 120M 等）
- 日线夜盘归属规则：凌晨(00:00-02:30)归属前一自然日

**批量聚合函数 `aggregate_1m_to_period()`：**
- 将历史 1M DataFrame 聚合为目标周期
- 与实时派生器使用相同的时间对齐规则
- 确保回测和实盘产出的K线完全一致

---

### 4. 鉴权管理器

新增 `ssquant/data/auth_manager.py` 模块：

**工作流程：**
1. 首次调用 `verify_auth()` 时，将账号密码发送给 data_server
2. data_server 内部转发至 kanpan789.com 完成实际验证
3. 验证成功后缓存结果（整个运行周期内只验证一次）

**接口：**
- `verify_auth()` - 验证用户身份
- `is_authenticated()` - 检查是否已鉴权通过
- `get_auth_message()` - 获取鉴权结果消息
- `reset_auth()` - 重置鉴权状态

---

### 5. 本地复权模块（占位）

新增 `ssquant/data/local_adjust.py` 模块：

**设计思路：**
- data_server（远程服务器）只存储不复权(raw)数据
- 复权计算在 ssquant 框架本地完成

**当前状态：**
- 占位实现，直通原始数据（等效于不复权）
- 后续实现后复权算法后，只需修改本文件即可

## 🔧 功能增强

### 6. 精确时间范围和BAR线数量请求

**支持三种数据请求方式（按优先级）：**
1. 精确时间范围: `start_time`/`end_time` (如 '2026-02-11 09:00:00')
2. 日期范围: `start_date`/`end_date` (如 '2026-02-11')
3. BAR线数量: `limit` (如 500，获取最近500根K线)

**可以组合使用：**
- `start_date + limit`: 从某日期开始取N条
- `start_time + limit`: 从某时间开始取N条
- 仅 `limit`: 取最近N条

**涉及文件：**
- `api_data_fetcher.py` - get_futures_data() 新增参数
- `unified_runner.py` - 配置验证放宽
- `backtest_data.py` - 参数传递

---

### 7. K线周期增强

**支持任意分钟周期（>=60分钟）：**
```python
config = {
    'kline_period': '65m',   # 65分钟周期
    'kline_period': '120m',  # 120分钟周期（2小时）
}
```

**日线夜盘归属规则修复：**
- 凌晨夜盘 (00:00-02:30)：归属前一自然日
- 日盘 (09:00-15:15) + 夜盘 (21:00-23:59)：归属当天

即：一个交易日 = 当天09:00-15:15 + 当天21:00-23:59 + 次日00:00-02:30

**新增函数：**
- `get_next_trading_day()` - 获取下一个交易日
- `_normalize_period()` - 周期格式标准化

---

### 8. CTP 断线重连优化

**断线检测与提示：**
- 区分**首次连接**和**断线重连**
- 连接断开时显示具体原因：
  - `0x1001`: 网络读取失败
  - `0x1002`: 网络写入失败
  - `0x2001`: 接收心跳超时
  - `0x2002`: 发送心跳超时
  - `0x2003`: 收到错误报文

**认证失败自动重试：**
- 重试间隔：3s → 6s → 9s → ... → 最长 30s
- 连接断开时自动停止重试
- 认证成功后重置计数器

**下单前连接状态检查：**
```
❌ [下单失败] CTP客户端未就绪！
   - 行情连接: 已断开
   - 交易连接: 已断开
```

## 🔄 升级指南

1. 更新 ssquant 包：
   ```bash
   pip install ssquant==0.4.2 --upgrade
   ```

2. 如需使用 data_server 模式，需额外安装：
   ```bash
   pip install websocket-client
   ```

3. Linux 用户首次使用需确保 CTP 库路径正确：
   ```bash
   # 如遇加载问题，可手动设置
   export LD_LIBRARY_PATH=/path/to/ssquant/ctp/py3xx:$LD_LIBRARY_PATH
   ```

4. 现有代码无需修改，完全向后兼容。

## ⚠️ 注意事项

- data_server 模式需要配合 data_server 服务端使用
- Linux CTP 仅支持 x86_64 架构
- 日线夜盘归属规则变更可能影响历史回测结果的一致性
- 建议在测试环境验证后再部署到生产环境
